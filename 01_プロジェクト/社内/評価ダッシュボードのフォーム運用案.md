
### 手順：フォームと新しいシートを紐付ける

1. **Googleフォームの編集画面**を開きます（回答用URLではなく、管理者が編集する画面）。
2. 上部の**「回答」タブ**をクリックします。
3. 「スプレッドシートにリンク」の右にある「︙（3点リーダー）」をクリックし、**「回答の保存先を選択」**を選びます。
    - （または緑色のスプレッドシートアイコンをクリックして「既存のスプレッドシートを選択」へ進みます）
4. **「既存のスプレッドシートを選択」**を選び、「選択」ボタンを押します。
5. 今回作成した**「新しい本番用スプレッドシート」**を選んで決定します。

### ⚠️ 重要：シート名の修正が必要です

この操作を行うと、新しいスプレッドシートの中に**新しいシート（タブ）**が自動生成されます（名前は恐らく「フォームの回答 2」などの番号付きになります）。

アプリは 

```
フォームの回答_自己
```

 という名前のシートを見に行く設定になっていますので、以下の修正をお願いします：

1. 新しい本番スプレッドシートを開く。
2. 元々あった空の（またはコピペした） 
    
    ```
    フォームの回答_自己
    ```
    
     シートを削除（または「_old」などにリネーム）する。
3. **フォーム連携によって新しく作られたシート**（「フォームの回答 2」など）の名前を、**
    
    ```
    フォームの回答_自己
    ```
    
    ** に変更する。

※部長評価用フォームについても同様の手順（保存先変更 → シート名を 

```
フォームの回答_部長
```

 に修正）を行ってください。

これで、「フォームに入力されたデータ」が「新しい本番シートの指定された名前のタブ」に自動的に入るようになり、アプリにも反映されるようになります。

### 2. 計算シート（

```
計算_部長
```

）の運用方法

この数式が入っているシート（恐らく

```
計算_部長
```

）も、アプリが「合計点」を表示するために必要不可欠です。 以下の手順で運用してください。

1. **計算シートもコピーする**: 古いスプレッドシートにある 
    
    ```
    計算_部長
    ```
    
     シートを、新しい本番用スプレッドシートに**コピー**してください。
    
2. **自動的に連動します**: この数式は 
    
    ```
    'フォームの回答_部長'
    ```
    
     という**「シート名」**を参照しています。 先ほどの手順で、新しいフォームの連携先シート名を 
    
    ```
    フォームの回答_部長
    ```
    
     にリネームしていただきました。 そのため、計算シートをコピーしてくるだけで、**自動的に新しいフォームの回答を参照して計算を始めてくれます**（数式の修正は不要です）。
    

**結論としての運用フロー:**

- 新しいスプレッドシートには、最終的に以下のシートが揃うことになります。
    1. ```
        評価マスターシート
        ```
        
        （コピーしてきたもの）
    2. ```
        フォームの回答_自己
        ```
        
        （フォームから自動生成＆リネーム）
    3. ```
        フォームの回答_部長
        ```
        
        （フォームから自動生成＆リネーム）
    4. ```
        計算_部長
        ```
        
        （コピーしてきたもの。これで自動計算される）

これで、アプリ側は全データを正しく取得できるようになります。
